# Contributing to Source Canon Faith

Thank you for your interest in contributing to Source Canon Faith. This project welcomes contributions that improve clarity, realism, and community safety.

## What We Welcome

### Text Improvements
- Clarify ambiguous passages
- Improve readability and accessibility
- Fix typos and grammatical errors

### Translation Improvements
- Better accuracy in existing translations
- Improved tone and cultural adaptation
- New language editions (contact maintainers first)

### Case Studies & Examples
- Real-world implementation stories
- Adapted examples for different cultural contexts
- Community feedback and outcomes

### Documentation
- Website improvements
- README enhancements
- Tutorial and guide creation

## Pull Request Process

1. **Open an issue** first for significant changes
   - Describe what you want to change and why
   - Explain how it improves clarity, realism, or safety

2. **Make your changes** in a feature branch
   - Use clear commit messages
   - Keep commits focused and atomic

3. **Submit a pull request**
   - Reference the issue number
   - Describe what changed and why
   - Be responsive to feedback

## Style Guidelines

- Write clear, accessible prose
- Avoid jargon where simple language works
- Be culturally sensitive and inclusive
- Maintain the core principles: dignity, justice, service, accountability

## Code of Conduct

We are committed to a respectful, inclusive community:

- Treat all contributors with respect
- Welcome diverse perspectives and backgrounds
- Focus on constructive feedback
- Prioritize community welfare over individual agendas

## Getting Help

- Open an issue for questions
- Check existing issues before duplicating
- Be patient with response times

---

*Every meaningful improvement deserves recognition. Thank you for helping make Source Canon Faith more accessible and practical for communities worldwide.*
